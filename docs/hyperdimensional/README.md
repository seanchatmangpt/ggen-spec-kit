## Key Concepts

### Hyperdimensional Information Theory (HDIT)

Mathematical framework for reasoning about semantic content in N-dimensional spaces where N >> human cognitive capacity (~7 items).

**Core Insight**: Software specifications exist in hyperdimensional semantic spaces. Traditional approaches project this to flat text, losing 99%+ information. HDIT preserves and analyzes the full semantic richness.

### The Constitutional Equation

```
spec.md = μ(feature.ttl)
```

Where μ is a deterministic, information-preserving transformation:
1. μ₁ Normalize (SHACL validation)
2. μ₂ Extract (SPARQL queries)
3. μ₃ Emit (Tera templates)
4. μ₄ Canonicalize (formatting)
5. μ₅ Receipt (cryptographic proof)

**Property**: H(feature.ttl) = H(spec.md) - no information loss.

### Information Theory Metrics

- **Shannon Entropy**: Measure ambiguity/uncertainty
- **KL Divergence**: Detect specification-implementation drift
- **Mutual Information**: Quantify feature coupling
- **Information Gain**: Prioritize dimensions/features

### Jobs To Be Done (JTBD) Integration

HDIT maps features to customer jobs and desired outcomes, enabling:
- Outcome-driven feature selection
- Customer value quantification
- Alignment measurement (spec ↔ job ↔ outcome)
